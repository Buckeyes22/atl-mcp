Caption (for the doc)

Every named failure mode in atl-mcp has a detection signal, a control that prevents or contains it, and a test that validates the control. The matrix groups them by category — security/integrity, correctness/idempotency, degraded operation, cross-cutting — and tags severity with a colored chip. fail-closed badges call out controls that abort the calling operation rather than degrade silently. Detail per row lives in the linked module docs and the threat model.
