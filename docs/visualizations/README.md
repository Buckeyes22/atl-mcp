# atl-mcp — Documentation Visualizations

> **What this is.** A static HTML site presenting the 18 visualizations that fill the visual-explanation gaps in atl-mcp's SDLC documentation. Open `index.html` in a browser. Every card links to a finished SVG you can lift into the corresponding markdown doc.
>
> **Output of:** the visualization analysis in [`/visualizations/README.md`](../../visualizations/README.md) (the brief), generated by Claude Design and brought into the project as a navigable artifact.

---

## How to view

```bash
# From repo root, pick one:
open docs/visualizations/index.html                          # macOS
xdg-open docs/visualizations/index.html                      # Linux
start docs/visualizations/index.html                         # Windows
python -m http.server 8000 --directory docs/visualizations   # local server
```

The site is fully static — no build step, no JS dependencies beyond CDN-loaded mermaid (used in a couple of visualizations) and Google Fonts (IBM Plex Sans / Mono). Works offline if those CDNs are unreachable, with degraded typography only.

## How to use the visualizations in markdown docs

Each visualization page contains:

1. The rendered **SVG figure** (or mermaid block — V1, V8 use mermaid).
2. A **caption** below the figure (already in the project voice; review before pasting).
3. **Design notes** explaining the visual language used.

To embed a figure in a markdown doc:

```html
<figure>
  <!-- paste the <svg>...</svg> from the page -->
  <figcaption>
    <!-- paste the caption from the page -->
  </figcaption>
</figure>
```

Or, simpler if your renderer supports it: link to the page directly.

```markdown
See [V1 — Audit chain entry construction](../visualizations/v01-audit-chain-construction.html).
```

## Editorial principles (from the design)

Quoted from the index page footer:

- Each figure must **replace** prose, not annotate it.
- If a reviewer asks the same question twice in a doc, that's a figure.
- Static SVG only — no interactivity, no JS dependencies. Versioned with the docs.
- One typeface (IBM Plex), one ink color, semantic palette consistent across all 18.
