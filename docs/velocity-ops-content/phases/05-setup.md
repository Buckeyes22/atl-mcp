# Phase 5: Project Setup & Kickoff

**Purpose:** Align on success, access, communication, and risk handling. Create project infrastructure.

## Required Inputs
- Executed contract set from Phase 4
- Deposit cleared

## Required Outputs
- Client-specific guardrails config from Phase 3 scope
- Business-intent test entries from Phase 2 discovery
- Deliverable checklist from scope document
- Single client-side owner identified
- Communication cadence agreed
- Project infrastructure created (repo, board, monitoring as needed)

## Guardrails
- **Single owner required** — one person on client side who can approve and unblock
- **Access method documented** — credential handling, environment access
- **Acceptance criteria set** — from SOW, translated into verifiable checks
- **Communication cadence defined** — varies by engagement shape (see 4H-12)

## Phase Boundary Checkpoint
**Setup → Architecture:** Is the project infrastructure ready? Does the team know what "done" looks like? If yes, proceed to Phase 6.
