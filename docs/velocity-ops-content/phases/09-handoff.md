# Phase 9: Launch, Handoff & Closeout

**Purpose:** Move safely into production, transfer ownership, capture proof, and set up ongoing relationship.

## Required Inputs
- Client UAT sign-off from Phase 8
- All acceptance criteria met

## Required Outputs
- Rollback plan executed or verified
- Handoff package complete (assets, credentials, documentation)
- Client acceptance per deliverable with sign-off
- Proof capture — case study draft, testimonial request, portfolio asset
- Consultant access narrowed or removed
- Final payment triggered by acceptance
- Referral ask (when relationship supports it)

## Guardrails
- **Rollback plan required before launch** — no launch without a recovery path
- **Handoff package completeness** — client has everything needed to operate independently
- **Proof capture is mandatory** — not optional. Feeds the proof asset pipeline.
- **Access offboarding** — consultant access narrowed/removed per security baseline

## Templates
- `templates/handoff-checklist.md`
- `templates/proof-asset-capture.md`
- `templates/uat-checklist.md`
- `templates/launch-checklist.md`

## Phase Boundary Checkpoint
**Handoff → Operations:** Does the client have everything they need to operate independently? Proof assets captured? Final payment received? If yes, proceed to Phase 10 (if ongoing relationship) or close.
