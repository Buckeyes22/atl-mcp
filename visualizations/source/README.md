# Agent Context Orchestrator MCP Server

> An MCP server that turns raw project requirements into agent-ready Jira, Confluence, and Bitbucket workspaces. The Jira project ([PCO](https://lateapexllc.atlassian.net/jira/software/projects/PCO/boards/1)) and Confluence space ([ACO](https://lateapexllc.atlassian.net/wiki/spaces/ACO)) accompanying this repo were generated by atl-mcp from atl-mcp's own requirements.
>
> **For reviewers: start at [`docs/demo/README.md`](docs/demo/README.md).**

> Status: production-quality through **M3**. M4-M11 tool surfaces remain feature-flagged until their milestone acceptance checks are complete.

Creates and maintains the Confluence + Jira + VCS context that implementation agents need to begin work with minimal ambiguity. Build-agent-agnostic — Codex, Claude Code, and any MCP-capable host are first-class consumers.

## Quick start

```bash
# Install
npm install

# Run tests (unit + integration + F-031 no-stdout lint)
npm test

# Typecheck
npm run typecheck

# Run the server (both stdio + Streamable HTTP transports + mgmt API)
npm run build && npm start

# Or via Docker
docker compose up
```

Once running:
- MCP Streamable HTTP on `http://localhost:3000/mcp` (POST + GET + DELETE per MCP spec).
- Mgmt API on `http://127.0.0.1:3001` (`/healthz`, `/readyz`, `/metrics`).
- Stdio: pipe a JSON-RPC client into the process.

## Architecture at a glance

- **Dual-port** ([open-edison F-130](docs/partners/open-edison.md)): MCP on 3000, mgmt API on 3001 (loopback only).
- **Dual-transport**: stdio (for embedded use) + Streamable HTTP ([indxr F-051](docs/partners/indxr.md): SSE keep-alive 25s, 1h sliding TTL, 1000 max concurrent).
- **File-only logger** ([simple-commands-mcp F-031](docs/partners/simple-commands-mcp.md)): pino → file. The runtime contract is "never stdout" — enforced by `npm run lint:no-stdout` and a vitest test.
- **Session capability registry** with two diagnostic resources: `orchestrator://session/current/capabilities` + `orchestrator://session/current/preflight` (auto-pinned per [mengram F-053](docs/partners/mengram.md); preflight payload becomes meaningful in M2).
